# MVP Specification

## 基本構成

- 1人用ゲームとする。
- タイトル画面をクリックまたはタップするとゲームを開始する。
- 小便器は8基固定とする。
- 男性NPCは3秒ごとに出現する。
- 待機人数は内部的には無制限とし、画面には先頭8人を表示する。

## NPCと便器の割り当て

- 尿量ゲージは4段階とする。
- 尿量が多いほど便器の使用時間を長くする。
- プレイヤーは空いている便器をクリックして指定する。
- 選択中の便器周辺を少し光らせる。
- 埋まっている便器は選択できない。
- NPCがNyoiceラインを越えた時点で目的地を確定する。
- 便器が未選択の場合、入口から最も近い空き便器へ自動配置する。
- 一度確定した目的地は変更しない。
- 使用後は左側の別出口から退出する。
- NPCが便器から離れた時点で、その便器を空き扱いにする。

## 不快ゲージとゲームオーバー

- 不快ゲージはトイレ全体で共有する。
- 隣接ペア数と時間経過に応じて不快ゲージを増加させる。
- 隣接ペア数が多いほど増加速度を上げる。
- 隣接が解消した時点で増加を停止する。
- 不快ゲージは減少しない。
- 不快ゲージがMAXになるとゲームオーバーとする。

## スコアとクリア

- NPCがExitへ到達した時点で、基本点100点に現在のコンボ倍率を掛け、`Mathf.RoundToInt()`で整数化した値を加点する。
- 隣接なしの状態が5秒続くごとにスコア倍率を`×1.0`、`×1.5`、`×2.0`、`×2.5`、`×3.0`の固定段階で上げる。
- コンボ時間は、最初のNPCが便器の利用地点へ到着して`UsingUrinal`へ遷移した時点から計測する。それ以前は`×1.0`のままとする。
- 隣接が発生した時点でコンボをリセットする。
- MVPではNPCを100人処理するとクリアとする。
- クリア表示後にタイトルへ戻る。
